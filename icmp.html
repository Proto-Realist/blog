ICMP Basics

Introduced in 1981, the Internet Control Message Protocol (ICMP) is a fundamental part of the TCP/IP suite. Because of its automated messages that provide feedback about a packets journey, it is essential for network diagnostics. It is most commonly known for its error reporting messages, like 'Desitination Unreachable' and its utilization within diagnostic tools like Ping and TraceRoute.

Ping takes advantage of ICMPs Echo Request and Echo Reply feature to see if a host reachable. TraceRoute uses its Time To Live (TTL) feature to trace the journey of a packet to and from a destination. ICMP has other features that The messages have a naming scheme in the type and code format. Here are some examples.

  • Echo Request (Type 8)
  • Echo Reply (Type 0)
  • Time Exceeded (Type 11)
  • Parameter Problem (Type 12)
